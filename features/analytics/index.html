    Analytics & Insights¶

    AI-powered insights and usage analytics for data-driven decision making.

    Overview¶

    SQLatte provides two analytics systems:

    1. Query Analytics - Track usage, performance, history
    2. Insights Engine - AI-powered data analysis

    Insights Engine¶

    What It Does¶

    Automatically analyzes query results and generates actionable insights.

    Example Query: "Show daily sales last 7 days"

    Insights Generated: - üìà Trend: Sales increased 15% over the week - ‚ö†Ô∏è Anomaly: Thursday had unusually low sales (holiday) - üí° Recommendation: Focus marketing on slow days

    Enable Insights¶

    insights:
  enabled: true
  mode: "hybrid"              # llm_only, statistical_only, hybrid
  max_insights: 3             # Per query
  include_statistical: true

    Insight Modes¶

    LLM Only - Pure AI analysis 

    mode: "llm_only"
    - ‚úÖ Creative, context-aware - ‚úÖ Natural language explanations - ‚ö†Ô∏è Higher LLM cost - ‚ö†Ô∏è Slower (1-2 seconds)

    Statistical Only - Rule-based 

    mode: "statistical_only"
    - ‚úÖ Fast (<100ms) - ‚úÖ No LLM cost - ‚úÖ Consistent results - ‚ö†Ô∏è Less context-aware

    Hybrid - Best of both (Recommended) 

    mode: "hybrid"
    - ‚úÖ AI + statistical analysis - ‚úÖ Balanced cost/quality - ‚úÖ Fallback to statistical if LLM fails

    Insight Types¶

    Trend - Patterns over time 

    üìà Sales increased 25% compared to last month

    Anomaly - Unexpected values 

    ‚ö†Ô∏è Login attempts 10x higher than average (possible attack)

    Summary - Key takeaways 

    üìä Top 3 products account for 60% of revenue

    Recommendation - Action items 

    üí° Consider restocking Product A (low inventory, high demand)

    Query Analytics¶

    What Gets Tracked¶

    Query History: - User questions - Generated SQL - Execution time - Rows returned - Success/failure - Timestamp

    Performance Metrics: - Average execution time - Query volume - Success rate - Popular tables - User activity

    Enable Analytics¶

    analytics:
  enabled: true
  storage: "postgresql"       # or "memory"

  postgresql:
    host: "localhost"
    port: 5432
    database: "sqlatte_analytics"
    user: "sqlatte"
    password: "password"

  query_history_days: 90      # Retention

    Analytics Dashboard¶

    Access: http://localhost:8000/analytics

    Metrics Cards: - Total queries executed - Average execution time - Success rate (%) - Active users (if auth enabled)

    Charts: - Queries per day/hour - Execution time trends - Top 10 queries - Table usage heatmap - Success vs failure ratio

    Example Queries¶

    Most popular queries: 

    SELECT question, COUNT(*) as count
FROM query_history
WHERE created_at > NOW() - INTERVAL '7 days'
GROUP BY question
ORDER BY count DESC
LIMIT 10;

    Slow queries: 

    SELECT question, sql, execution_time_ms
FROM query_history
WHERE execution_time_ms > 5000
ORDER BY execution_time_ms DESC
LIMIT 20;

    Daily query volume: 

    SELECT 
  DATE(created_at) as date,
  COUNT(*) as queries,
  AVG(execution_time_ms) as avg_time
FROM query_history
GROUP BY DATE(created_at)
ORDER BY date DESC
LIMIT 30;

    Configuration¶

    Full Configuration¶

    insights:
  enabled: true
  mode: "hybrid"
  max_insights: 3
  include_statistical: true

  # Advanced settings
  min_confidence: 0.7         # Filter low-confidence insights
  context_window: 100         # Rows to analyze
  llm_timeout: 10             # Seconds

analytics:
  enabled: true
  storage: "postgresql"

  postgresql:
    host: "postgres"
    port: 5432
    database: "sqlatte_analytics"
    user: "sqlatte"
    password: "${ANALYTICS_PASSWORD}"

  # Retention
  query_history_days: 90
  execution_logs_days: 30

  # Performance
  batch_size: 100
  flush_interval_seconds: 60

    Best Practices¶

    Insights¶

    • ‚úÖ Enable hybrid mode for best results
    • ‚úÖ Set max_insights: 3 to avoid overwhelming users
    • ‚úÖ Review insights for accuracy before relying on them
    • ‚ö†Ô∏è Don't use insights as sole decision-making tool

    Analytics¶

    • ‚úÖ Use PostgreSQL storage for production
    • ‚úÖ Set appropriate retention (30-90 days)
    • ‚úÖ Monitor database size regularly
    • ‚úÖ Create indexes for performance
    • ‚ö†Ô∏è Don't track sensitive data (enable anonymization if needed)

    Performance¶

    • ‚úÖ Use statistical mode for real-time dashboards
    • ‚úÖ Cache insights for frequently-run queries
    • ‚úÖ Limit context window for large result sets
    • ‚ö†Ô∏è LLM insights add 1-2 seconds to response time

    Privacy & Compliance¶

    Anonymize Data¶

    analytics:
  anonymize_users: true       # Hash user IDs
  anonymize_queries: false    # Keep queries readable
  anonymize_results: false    # Keep results readable

    GDPR Compliance¶

    Delete user data: 

    DELETE FROM query_history WHERE user_id = 'user123';
DELETE FROM execution_logs WHERE user_id = 'user123';

    Export user data: 

    SELECT * FROM query_history 
WHERE user_id = 'user123'
ORDER BY created_at DESC;

    Troubleshooting¶

    No Insights Generated¶

    # Check insights are enabled
insights:
  enabled: true

# Check LLM provider is working
curl http://localhost:8000/health

# Check logs for errors
docker-compose logs -f sqlatte | grep insights

    Analytics Dashboard Empty¶

    # Verify analytics enabled
analytics:
  enabled: true

# Check database connection
docker-compose exec postgres psql -U sqlatte -d sqlatte_analytics

# Check tables exist
\dt

# Check data exists
SELECT COUNT(*) FROM query_history;

    Slow Performance¶

    # Add database indexes
CREATE INDEX idx_query_history_created ON query_history(created_at);
CREATE INDEX idx_query_history_user ON query_history(user_id);

# Reduce retention
analytics:
  query_history_days: 30  # Instead of 90

# Use statistical mode only
insights:
  mode: "statistical_only"

    Monitoring¶

    Database Size¶

    -- Total size
SELECT pg_size_pretty(pg_database_size('sqlatte_analytics'));

-- Table sizes
SELECT 
  tablename,
  pg_size_pretty(pg_total_relation_size(tablename::regclass))
FROM pg_tables
WHERE schemaname = 'public';

    Query Volume¶

    -- Today's queries
SELECT COUNT(*) FROM query_history 
WHERE created_at > CURRENT_DATE;

-- Last 7 days by day
SELECT 
  DATE(created_at) as date,
  COUNT(*) as queries
FROM query_history
WHERE created_at > CURRENT_DATE - INTERVAL '7 days'
GROUP BY DATE(created_at);

    Limitations¶

    Insights Engine: - ‚ö†Ô∏è Works best with time-series data - ‚ö†Ô∏è Limited to numeric/date columns for trends - ‚ö†Ô∏è May miss complex patterns - ‚ö†Ô∏è Requires sufficient data (minimum 5-10 rows)

    Analytics: - ‚ö†Ô∏è PostgreSQL required for persistence - ‚ö†Ô∏è Not shared across multiple instances (use shared DB) - ‚ö†Ô∏è No real-time streaming (batch updates)
